# Legal and operations guardrails

이 문서는 제품 운영 체크리스트이며 법률자문이 아닙니다. 유료 출시 전에는 특히 민간 플랫폼, 네이버 API, 개인위치정보 처리 여부를 전문 검토해야 합니다.

## 원칙

1. 공식 OpenAPI/RSS/제휴 피드를 HTML보다 우선합니다.
2. `robots.txt` 허용은 접근권·복제권·재게시 허락이 아닙니다. [RFC 9309](https://www.rfc-editor.org/rfc/rfc9309.html)과 별개로 이용약관·저작권·라이선스를 확인합니다.
3. 제목, 날짜, 장소, 대상, 가격, 상태, 공식 URL 같은 행사 사실 중심으로 저장합니다. 명시적 허락이 없으면 본문 전체·포스터·첨부파일을 복제하지 않습니다.
4. 신청/결제는 항상 주최기관 원문에서 수행합니다.
5. 403·캡차·로그인·리디렉션 루프·NetFunnel이 보이면 우회하지 않고 소스를 중단합니다.

## source registry에 필요한 필드

```text
source_id, owner, base_url, source_type(api|rss|html|submission)
official, terms_url, copyright_url, license_code
robots_url, robots_status, robots_checked_at, robots_hash
allowed_paths, denied_paths, rate_limit, contact
parser_version, last_success_at, legal_review_status, enabled
```

이 MVP의 `SourceInfo`는 핵심 일부만 포함합니다. 운영 전에는 나머지를 DB 테이블로 승격하고, robots/정책 재검토 만료일을 둡니다.

## 저작권·공공데이터

- [공공데이터 이용정책](https://www.data.go.kr/ugs/selectPortalPolicyView.do)은 제3자 권리가 섞인 경우 별도 허락이 필요하다고 안내합니다.
- [공공누리 유형](https://www.kogl.or.kr/info/license.do)은 상업 이용과 변경 허용 범위가 유형별로 다릅니다. 레코드마다 `license_code`와 출처를 보존합니다.
- 데이터베이스의 개별 항목도 반복적·체계적으로 복제하면 데이터베이스 제작자의 권리와 충돌할 수 있습니다. 공개 화면이라는 이유만으로 민간 카탈로그를 전량 복제하지 않습니다.
- 썸네일 URL을 받더라도 이미지 파일의 재배포 권리가 자동으로 생기지 않습니다. 기본 UI는 원문 링크와 출처 표기를 우선합니다.

## 개인정보·위치

- 커뮤니티 작성자명, 프로필, 전화번호, 채팅 원문/캡처, 아이 사진은 수집하지 않습니다.
- 주최기관의 공개 대표 전화와 개인 담당자 연락처를 구분합니다. 개인 연락처는 알림 본문에서 제외하는 편이 안전합니다.
- 부모의 위치는 동네 중심점 또는 기기 내 반경 검색을 우선합니다. 아이 이름·학교·정확한 생년월일·이동경로는 저장하지 않습니다.
- 정확한 위치를 서버에 지속 저장하면 [위치정보법](https://www.law.go.kr/LSW/lsInfoP.do?lsiSeq=277359)상 동의·약관·보유기간·사업 신고/허가 이슈를 별도 검토합니다.
- [개인정보보호법](https://law.go.kr/LSW/lsInfoP.do?lsiSeq=270351)에 따라 목적에 필요한 최소 데이터만 보유하고 삭제 정책을 둡니다.

## HTTP 운영 규칙

- 실명형 User-Agent와 모니터링되는 운영자 연락처
- HTML은 도메인당 동시성 1, 기본 요청 간격 5초 이상(일일 실행은 가능하면 목록 1회 GET)
- `ETag`, `Last-Modified`, `If-Modified-Since`, sitemap/RSS 우선
- 429 `Retry-After` 준수, 일시 오류 지수 백오프
- 401/403/캡차/WAF/로그인 리디렉션은 재시도 우회 없이 중단
- robots의 명확한 HTTP 404/410은 RFC 9309 §2.3.1.3에 따라 규칙 없음으로 처리하고, 5xx·네트워크 오류·WAF HTML은 전체 차단으로 처리
- robots 파서는 origin별로 캐시하되 허용 여부는 요청 경로마다 다시 계산. 공개 경로 확인 결과를 같은 호스트의 비공개 경로에 재사용하지 않음
- 리디렉션은 한 단계씩 검사하고 다른 origin으로 이동하면 중단. 같은 origin의 새 경로도 robots 규칙을 다시 평가
- 요청/응답 로그에서 API 키·쿠키·Authorization 제거
- 원문 삭제·취소·권리자 요청을 24시간 이내 검색/알림에서 제거

현재 `PoliteHttpClient`는 호스트 간격, 재시도, `Retry-After`, 경로별 RFC 9309 robots 판정, 안전한 수동 리디렉션, 운영체제 인증서 저장소 검증을 제공하고, HTTP 오류에는 호스트만 남겨 URL 경로·쿼리의 키가 수집 이력에 들어가지 않게 합니다. 조건부 GET 캐시와 구조화 로그의 필드별 마스킹은 다음 단계입니다.

민간 공개 목록은 기본 비활성만으로 끝내지 않는다. 정책 검토를 통과한 정확한 source ID를 쉼표로 구분해 `KIDS_RADAR_APPROVED_SOURCES`에 넣어야 `available()`이 참이 된다. 따라서 `kidradar crawl --all`도 승인되지 않은 민간 소스에 네트워크 요청을 보내지 않는다. 공개 공공기관 목록은 별도 민간 승인 없이 선택 실행할 수 있지만, 실행 시점의 robots 검사는 동일하게 통과해야 한다.

삼성 이노베이션 뮤지엄처럼 robots URL이 유효한 정책 파일 대신 모호한 HTML을 반환하는 소스는 정책 승인과 별도로 `KIDS_RADAR_ROBOTS_OVERRIDE_SOURCES`의 정확한 source ID를 요구한다. 이 값은 운영자가 모호성을 인지했다는 기록일 뿐 접근·저작권 허가가 아니며, 명시적 `Disallow`나 WAF를 무력화하지 않는다.

### 차단 소스의 공식 대체면

원래 호스트가 전면 차단되거나 점검 중이어도, 동일 기관 또는 감독 지자체가 별도로 공개한 API·통합예약 목록·보도자료·RSS가 있으면 그 **별도 공개면 자체**를 새 수집원으로 검토할 수 있다. 원래 차단은 그대로 유지하며 다음 조건을 모두 지킨다.

- 대체면의 운영 주체와 원문 출처가 공식 기관임을 확인
- 대체면의 robots와 이용조건을 원래 호스트와 별도로 검사
- 차단된 원래 호스트, 이미지, 첨부, 신청·로그인 링크는 후속 호출하지 않음
- 대체면이 제공하는 사실 메타데이터만 저장하고 전체 일정 커버리지로 과장하지 않음
- 레지스트리와 감사 문서에 원래 차단 상태와 대체면 URL을 함께 기록

`goyang_children_museum_city_news`가 이 패턴이다. 고양어린이박물관 원래 도메인은 계속 미호출하고, 고양시 뉴스가 직접 공개한 기사 일정만 구조화한다. 수원도서관은 차단 우회가 아니라 같은 기관의 점검 중인 구형 주소를 정상 운영 중인 공식 통합예약 주소로 교체한 경우다.

## 발송 직전 검증

수집 시점의 `접수중`은 발송 시점에 마감일 수 있습니다.

1. 공식 API의 상태와 접수 마감 재조회
2. 상세 원문이 200이며 취소/마감 공지가 없는지 확인
3. 확인이 불가능하면 “잔여 여부는 원문 확인” 표시
4. 이미 알림한 동일 회차는 재발송하지 않음
5. 사용자의 반경·연령·요일·가격 필터를 발송 직전에 다시 적용

기본 조회와 다이제스트는 마지막 정상 관측 후 48시간이 지난 행을 제외한다. 수집 실패 중인 소스의 오래된 `접수중` 상태를 계속 알리는 대신 원문 재확인을 요구하며, 감사 목적에서만 `max_stale_hours`를 명시적으로 해제한다.

## 품질·정책 테스트 출시 기준

- parser fixture: JSON/XML/RSS/승인 HTML, 한국식 날짜와 `초1~3`, `무료(재료비 별도)`
- contract: 필수 필드·페이지네이션·좌표 범위·canonical URL. 필드 변경 시 조용한 누락 대신 소스 중단
- policy: 꿈길·카페·채팅·NetFunnel URL을 실제 요청 전에 차단
- network: 304, 429, 403, 5xx, timeout, redirect loop, HTML 오류 응답
- dedupe: 재게시·제목 약칭·날짜별 회차 골든셋. 자동 병합 정밀도 98% 이상 권장
- extraction: 날짜·가격·학년·장소 95% 이상, 필수 필드 정상률 99% 이상
- privacy: DB·로그·알림에 작성자/프로필/채팅/API 키가 없어야 함
- daily canary: 소스별 건수 급변, null 비율, 미래/과거 비율, 마지막 성공 시각
- deletion: 원문 삭제·취소·권리자 요청 24시간 이내 반영
