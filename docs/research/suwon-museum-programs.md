# 수원시 3개 박물관 프로그램 수집 근거

검증일: 2026-07-15 (KST)

## 결론

수원시박물관사업소 통합예약시스템은 로그인 없이 볼 수 있는 공식 프로그램
목록과 정보 상세를 제공한다. 목록 화면은 내부적으로 공개 POST JSON 조회를
사용하지만 상태를 바꾸지 않는 검색 요청이다. 구현은 다음 두 종류의 요청만
허용한다.

- 공개 목록: `POST https://rmuseum.suwon.go.kr/progrm/progrmAjaxList.do`
- 공개 정보 상세: `GET https://rmuseum.suwon.go.kr/progrm/progrmDetail.do`

로그인, 예약 신청, 결제, 대기열, 잔여석 변경, 개인정보 입력 경로는 호출하지
않는다.

## 기관별 소스

| 코드 | 기관 | source_id | 2026-07-15 현재 목록 |
|---|---|---|---:|
| `SW` | 수원박물관 | `suwon_museum_child_programs` | 1 |
| `GG` | 수원광교박물관 | `suwon_gwanggyo_museum_child_programs` | 2 |
| `HS` | 수원화성박물관 | `suwon_hwaseong_museum_child_programs` | 2 |

공개 목록은 `museumCd=GG`를 보내도 세 박물관 행을 함께 돌려주는 현행 동작이
확인됐다. 따라서 응답의 `museumCd`를 다시 정확히 비교해 기관 간 중복을 막는다.

현재 5개 상세의 공개 사실을 다시 읽은 결과는 다음과 같다.

- 수원화성박물관 2개: 참가대상 `가족,어린이`, 무료
- 수원박물관 1개: 참가대상 `어린이`, 5,000원
- 수원광교박물관 2개: 참가대상 `어린이` 또는 `어린이,유아`, 각 5,000원
- 목록 상태: 접수대기 1, 접수중 1, 접수마감 3

## 파싱 및 보존 범위

목록에서는 기관 코드, 숫자 프로그램 식별자, 제목, 진행 기간, 접수 기간,
접수 상태만 사용한다. 상세에서는 아래 라벨의 사실만 읽는다.

- 프로그램명
- 진행일시
- 접수일시
- 장소
- 참가대상
- 접수방법
- 접수방식
- 참가비
- 문의처

상세 하단에는 예약 현황과 마스킹된 신청자 표시가 함께 렌더링될 수 있다. 이
영역은 파싱하지 않으며 원문 HTML도 `raw`에 저장하지 않는다. 테스트 픽스처에
마스킹된 ID와 전화번호를 넣어 결과에 섞이지 않는 것을 확인한다.

`2026년 7월 11, 25일`, `2026.4.22 ~ 5.20`, 오전/오후 시각과 같은 실제 날짜
표현을 별도로 정규화한다. 식별자는 숫자만 허용하며 상세 URL은 코드와 식별자로
새로 조립한다.

## 정책과 운영 경계

- 기본 비활성화 opt-in 소스다.
- 저작물 재사용 조건을 단정할 수 없어 라이선스 값은 비워 둔다.
- 실행 시 목록과 각 상세에 robots 정책을 확인한다.
- 2026-07-15 검증 중 사이트 연결은 간헐적으로 타임아웃됐고 `/robots.txt`도
  한 차례 확인 실패했다. 이 경우 공통 HTTP 클라이언트는 수집을 허용하지 않고
  실패 종료한다. 목록/상세가 공개라는 사실로 robots 실패를 우회하지 않는다.
- 구조가 바뀌어 `dataList`나 상세의 프로그램명이 사라지면 빈 성공으로 처리하지
  않고 오류로 종료한다.
